## 使用方式

### 1. 逐模块开发
建议按以下顺序逐个模块开发：
```
瞭望管理 → 瞭望采集 → 数据仓库 → 深度采集 → 接口管理
```

原因：
- 瞭望管理是数据源配置基础
- 瞭望采集依赖瞭望管理的数据源
- 数据仓库存储瞭望采集的结果
- 深度采集处理数据仓库的数据
- 接口管理相对独立，可最后开发

### 2. 提示词使用方法
将对应的 `.md` 文件内容完整复制到 Trae IDE 的对话框中，作为开发指令。

### 3. 每个提示词包含的内容
- **角色设定**：定义AI开发者的角色
- **任务目标**：明确要实现的功能
- **技术栈**：项目使用的技术
- **功能需求**：详细的功能描述
- **数据库设计**：表结构定义
- **文件清单**：需要创建/修改的文件
- **路由注册**：URL路由配置
- **验收标准**：功能验证清单

## 开发注意事项

1. **数据库操作**：统一使用 `get_connection()` 上下文管理器
2. **返回格式**：JSON 统一格式 `{"success": true/false, "message": "...", "data": {...}}`
3. **代码风格**：参考项目中已有的 `admin_manage.py` 和 `user.py`
4. **路由注册**：在 `app.py` 中注册，注意与其他模块路由不冲突
5. **模板继承**：大部分页面继承 `base.html`，瞭望采集页面独立风格
